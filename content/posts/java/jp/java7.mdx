### 数字タイプ

今回は様々な数字タイプを詳しく見てみましょう。  
すぐに次のコードを書いてみましょう。

#### Var1
```java
package variable;

public class Var1 {
    public static void main(String[] args) {
        //整数
        byte b = 127; //-128 ~ 127
        short s = 32767; // -32,768 ~ 32,767
        int i = 2147483647; //-2,147,483,648 ~ 2,147,483,647 (約20億)

        //-9,223,372,036,854,775,808 ~ 9,223,372,036,854,775,807
        long l = 9223372036854775807L;

        //実数
        float f = 10.0f;
        double d = 10.0;
    }
}
```

メモリを少なく使用すれば小さな数字を表現でき、メモリを多く使用すれば大きな数字を表現できます。  
変数を宣言すると、表現範囲に応じてメモリ空間を占有します。そのため、必要に応じて様々なタイプを提供します。


表現できる数字の範囲と占有するメモリ空間は次のとおりです。その他のタイプも一緒に整理しておきました。  
- **整数型**
    - `byte` : -128 ~ 127 (1byte, 2⁸)
    - `short` : -32,768 ~ 32,767 (2byte, 2¹⁶)
    - `int` : -2,147,483,648 ~ 2,147,483,647 (約20億) (4byte, 2³²)
    - `long` : -9,223,372,036,854,775,808 ~ 9,223,372,036,854,775,807 (8byte, 2⁶⁴)  
- **実数型**
    - `float` : 大体 -3.4E38 ~ 3.4E38、7桁精度 (4byte, 2³²)
    - `double` : 大体 -1.7E308 ~ 1.7E308、15桁精度 (8byte, 2⁶⁴)  
- **その他**
    - `boolean` : `true`、`false` (1byte)
    - `char` : 文字一つ(2byte)
    - `String` : 文字列を表現します。メモリ使用量は文字の長さに応じて動的に変わります。

#### リテラルタイプ指定
- 整数リテラルは`int`を基本として使用します。したがって、`int`範囲まで表現できます。数字が`int`範囲である約20億を超える場合は、`L`を付けて整数リテラルを`long`に変更する必要があります。（大文字`L`、小文字`l`の両方が可能ですが、小文字`l`は数字1と混同する可能性があるため推奨されません。）
- 実数リテラルは基本が`double`型を使用します。`float`型を使用する場合は、`f`を付けて`float`型として指定する必要があります。

### 変数タイプ整理

#### 次のタイプは実務でほとんど使用しません。
- `byte` : 表現長さが小さすぎます。また、Javaは基本として4byte(`int`)を効率的に計算するように設計されています。`int`を使用しましょう。
    - `byte`タイプを直接宣言し、ここに数字値を代入して計算する作業はほとんどありません。
    - 代わりにファイルをバイト単位で扱うため、ファイル転送、ファイルコピーなどに主に使用されます。
- `short` : 表現長さが小さすぎます。また、Javaは基本として4byte(`int`)を効率的に計算するように設計されています。`int`を使用しましょう。
- `float` : 表現長さと精度が低いです。実数型は`double`を使用しましょう。
- `char` : 文字一つを表現する作業はほとんどありません。文字一つを表現する際も文字列を使用できます。
    - 例えば`String a = "b"`のように使用すればよいです。


#### よく使用するタイプ
実務でよく使用するタイプは次のとおりです。  
- 整数 - `int`、`long` : Javaは整数に基本として`int`を使用します。もし20億を超える可能性がある場合は`long`を使用すればよいです。
    - ファイルを扱う際は`byte`を使用します。
- 実数 - `double` : 実数は悩まず`double`を使用すればよいです。
- ブール型 - `boolean` : `true`、`false`で真偽を表現します。以降の条件文でよく使用されます。
- 文字列 - `String` : 文字を扱う際は、文字一つでも文字列でもすべて`String`を使用するのが便利です。

よく使用するタイプを除いて、実務で残りを使用する場合はほとんどありません。せいぜいファイル転送時に`byte`を使用する程度です。

## 変数命名規則

Javaで変数の名前を付ける際には規則と慣例があります。  
**規則は必須**です。規則を守らないとコンパイルエラーが発生します。  
**慣例は必須ではありませんが、世界中の開発者がその慣例に従うため、事実上規則**と考えてもよいです。

#### 規則
- 変数名は数字で始めることはできません。（例: `1num`、`1st`）
    - しかし、数字を名前に含めることは可能です（例: `myVar1`、`num1`）。
- 名前には空白を含めることはできません。
- Javaの予約語を変数名として使用することはできません。（例: `int`、`class`、`public`）
- 変数名には英文字(`a-z`、`A-Z`)、数字(`0-9`)、ドル記号(`$`)またはアンダースコア(`_`)のみ使用できます。

#### 慣例
- 小文字で始めるキャメルケース
- 変数名は小文字で始めるのが一般的です。複数の単語で構成された変数名の場合、最初の単語は
- 小文字で始め、その後の各単語は大文字で始めるキャメルケース(camel case)を使用します。（例: `orderDetail`、`myAccount`）

**キャメルケース**  
キャメルケース(Camel Case)はプログラミングで変数、関数、クラスなどの名前を付ける際によく使用する表記法の一つです。この表記法の名前は、書いた名前が複数の単語で構成されている場合、各単語の最初の文字が大文字で始まり、この大文字がラクダのこぶのように見えることから由来しました。キャメルケースを使用すると、名前に空白を入れなくても複数の単語を簡単に区別できるため、変数の名前を理解しやすくします。また、ほとんどのプログラミング言語では名前に空白を含めることができないため、キャメルケースはこのような制限を回避する良い方法です。

### Java言語の慣例を一度に整理
### クラスは大文字で始め、残りは小文字で始める

- Javaでクラス名の最初の文字は大文字で始めます。そして残りはすべて最初の文字を小文字で始めます。ここにキャメルケースを適用すればよいです。このようにすれば、すべてのJava慣例を覚えることができます！
- 例: クラスは最初の文字大文字、残りはすべて最初の文字小文字で始める + キャメルケース
    - クラス: `Person`、`OrderDetail`
    - 変数を含む残り: `firstName`、`userAccount`
- ここに例外がちょうど2つあります。
    - 定数はすべて大文字を使用し、アンダースコアで区別します。（定数は後で学習）
        - `USER_LIMIT`
    - パッケージはすべて小文字を使用します。（パッケージは後で学習）
        - `org.spring.boot`

**参考** : 変数名は意味があり、その用途を明確に説明する必要があります。
- `a`、`b` : このような変数は用途を説明しません。単純な例でのみ使用するのがよいです。
- `studentCount`、`maxScore`、`userAccount`、`orderCount` : 用途を明確に説明します。
